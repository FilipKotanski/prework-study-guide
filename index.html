HTML

  • head element contains information about the webpage.
  • body element represents the visible content shown to the user.
  • h1 to h6 elements define headings of different levels, with h1 being the highest level and h6 the lowest.
  • p element represents a paragraph of text.
  • a element creates a hyperlink to another web page, document, or a specific location within the same page.
  • img element inserts an image into the document.
  • ul element defines an unordered (bulleted) list.
  • ol element defines an ordered (numbered) list.
  • li element represents an item in a list (ul or ol).
  • div element defines a division or a container for grouping and styling other elements.
  • span element defines an inline container for a small section of text or other inline elements.
  • table element creates a table for organizing data into rows and columns.
  • tr element defines a table row.
  • td element represents a table cell within a tr element
  • th element defines a table header cell.
  • form element creates a form for user input, such as text fields, checkboxes, and submit buttons.
  • input form represents an input field within a form.
  • button element creates a clickable button.
  • textarea element defines a multi-line text input control.
  • select element creates a dropdown list or a list box.
  • optgroup element groups related option elements within a select element.
  • label element provides a text label for an input, select, or textarea element.
  • iframe element embeds an inline frame or an external web page within the document.
  • audio element embeds audio content, such as music or sound clips.
  • video element embeds video content, such as movies or video clips.
  • script element inserts client-side scripts, typically JavaScript, into the document.
  • style element contains CSS (Cascading Style Sheets) code for styling the document.
  • meta element specifies metadata about the document, such as character encoding, viewport settings, or keywords.

CSS

  • Selectors: Used to select the HTML elements you want to style.
  • Box Model: Describes the design and layout of elements in CSS.
  • Layouts (Flexbox and Grid): Allows for flexible and grid-based layouts.
  • Responsive Design (Media Queries): Adapts styles for different devices and screen sizes.
  • Transitions and Animations: Creates animated effects between different CSS styles.
  • Typography (Google Fonts, @font-face): Customizes text styles and integrates custom fonts.
  • Backgrounds and Gradients: Styles backgrounds with color, image, and gradient options.
  • Transforms and Transforms 3D: Modifies the shape, size, and position of elements in 2D or 3D space.
  • Pseudo-classes and Pseudo-elements: Styles based on various states or positions of elements.
  • Variables (CSS Custom Properties): Stores and reuses values for easier style management.
  • Box Shadow and Text Shadow: Adds shadows to elements for depth and dimension.
  • Responsive Units (e.g., vw, vh, %, em, rem): Allows for responsive layouts and scalable designs.

Git

  • git init: Initializes a new Git repository.
  • git clone: Clones a repository into a new directory.
  • git add: Adds changes in the working directory to the staging area.
  • git commit: Records changes to the repository with a message.
  • git push: Uploads local repository content to a remote repository.
  • git pull: Fetches and merges changes from a remote repository to a local repository.
  • git branch: Lists, creates, or deletes branches.
  • git merge: Combines branches into the current branch.
  • git checkout: Switches branches or restores working tree files.
  • git status: Displays the status of the working directory and the staging area.
  • git log: Shows the commit logs.
  • git remote: Manages set of tracked repositories.

JavaScript

  • Variables and Data Types: Used to store and manipulate data.
  • Functions: Blocks of code designed to perform a certain task.
  • Arrays: Used to store multiple values in a single variable.
  • Objects: Used to store collections of data and more complex entities.
  • Control Flow Statements: Used to control the flow of the program based on conditions.
  • Loops: Used to execute a block of code multiple times.
  • Events: Used to trigger specific functionality based on user interactions.
  • DOM Manipulation: Used to interact with the Document Object Model (DOM) of the web page.
  • AJAX (Asynchronous JavaScript and XML): Used to update parts of a web page without reloading the whole page.
  • ES6 Features (Arrow functions, Classes, Promises): Provides new syntax and features to make JavaScript more efficient and concise.
  • Error Handling: Used to handle errors and exceptions in JavaScript code.
  • JSON (JavaScript Object Notation): Used for storing and exchanging data.
